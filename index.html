Birthday Paradox Simulator

What Is The Birthday Paradox?

The birthday paradox is a probability puzzle stating that in a group of just 23 people, there's a greater than 50% chance that two people share the same birthday.

How Does It Work?

The program generates a list of random birthdays and tests for duplicates across multiple experiments. Each experiment increases the number of birthdays by five, running 20 simulations per batch from 5 to 100 people.

Implementation

I implemented the Trial, Experiment, and Simulator classes, following SOLID OOP principles and writing thorough unit tests for functionality.

What I Learned

This was my first experience applying SOLID principles and test-driven development. It also taught me how to use GitHub collaboratively with proper commits, branches, and code reviews.

Elevator Simulator

What Is It?

A simulation of elevator behavior using different types (Standard, Freight, Express, Penthouse) and schedulers (Simple, Complex) that respond to building calls.

How It Works

The program takes arguments such as the number of floors, elevator types, and a call file. Elevators respond to calls based on their specific rules, moving one floor per round.

Design

Elevators inherit from an abstract class defining shared logic. Schedulers implement a common interface but apply different strategies. Calls are modeled as objects, improving readability and debugging.

Implementation

I built and tested elevator logic, implemented the Complex Scheduler, and helped refactor the Call class system. I also resolved merge conflicts and improved code quality.

What I Learned

This project emphasized test-driven development, abstraction, inheritance, and collaborative Git workflows including conflict resolution and proper use of pull requests.
